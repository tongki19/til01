### Context

파이썬(Python)에서 리스트(List)의 요소를 필터링하는 방법은 크게 두 가지가 있다. 하나는 리스트 컴프리헨션(List Comprehension)을 사용하는 것이고, 다른 하나는 `filter()` 함수를 사용하는 것이다. 리스트 컴프리헨션은 파이썬다운(Pythonic) 코드 작성 방식으로 알려져 있지만, 조건문이 여러 개 중첩되거나 복잡해지면 가독성이 떨어질 수 있다. 이 때문에 일부 개발자들은 가독성을 위해 `filter()` 함수를 선호하기도 한다. 이번 학습은 가독성과 성능 사이에서 어떤 선택을 해야 하는지에 대한 고찰을 담고 있다.

### Core

리스트 컴프리헨션은 `[x for x in data if condition1 and condition2]`와 같은 형태로 작성되며, `filter()` 함수는 `list(filter(lambda x: condition1 and condition2, data))` 형태로 사용된다. 아래는 대규모 데이터 처리 시 두 방법의 성능 차이를 비교하는 벤치마크 코드다.

```python
import timeit
import random

# 100만 개의 무작위 숫자로 이루어진 리스트 생성
setup_code = """
import random
data = [random.randint(1, 1000) for _ in range(1000000)]
"""

# 조건: 100보다 크고 짝수인 숫자만 필터링
condition_comp = "[x for x in data if x > 100 and x % 2 == 0]"
condition_filter = "list(filter(lambda x: x > 100 and x % 2 == 0, data))"

# 벤치마크 실행 (각각 10회 반복)
time_comp = timeit.timeit(stmt=condition_comp, setup=setup_code, number=10)
time_filter = timeit.timeit(stmt=condition_filter, setup=setup_code, number=10)

print(f"List Comprehension Time: {time_comp:.6f} seconds")
print(f"filter() Function Time: {time_filter:.6f} seconds")

# 실행 결과 (예시)
# List Comprehension Time: 0.817200 seconds
# filter() Function Time: 1.054321 seconds
```

### Insight

위 벤치마크 결과에서 볼 수 있듯이, 일반적으로 리스트 컴프리헨션이 `filter()` 함수를 사용하는 것보다 더 빠르다. 이는 CPython 구현에서 리스트 컴프리헨션이 C 언어로 최적화되어 구현된 반면, `filter()` 함수는 내부적으로 람다 함수 호출과 같은 파이썬 객체의 오버헤드가 발생하기 때문이다.

따라서 코딩 스타일을 결정할 때 다음과 같은 기준을 적용할 수 있다.

*   **가독성 우선 (비전공자/협업 환경):** 조건이 복잡할 경우, `filter()`와 명명된(named) 함수를 조합하여 조건 로직을 별도로 분리하면 코드의 의도가 명확해진다.
*   **성능 우선 (대규모 데이터 처리):** 대량의 데이터를 처리해야 하거나 미세한 속도 차이가 중요한 상황에서는 리스트 컴프리헨션을 사용하는 것이 유리하다.
