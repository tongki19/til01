### Context

파이썬에서 리스트의 요소를 필터링할 때, 리스트 컴프리헨션(List Comprehension)은 강력하고 간결한 문법을 제공합니다. 그러나 조건이 여러 개 중첩되거나 복잡해질 경우 `[x for x in list if condition1 if condition2]`와 같은 형태는 가독성을 떨어뜨릴 수 있습니다. 이로 인해 `filter()` 함수를 사용하는 것이 코드의 명확성을 높이는 대안으로 고려되곤 합니다. 특히 비전공자나 협업하는 팀원에게 코드를 이해하기 쉽게 전달해야 할 때 이러한 고민이 발생합니다.

### Core

두 방법의 코드 구조와 성능을 비교하기 위해 대규모 데이터를 필터링하는 예시 코드를 작성합니다. `timeit` 모듈을 사용하여 두 방식의 실행 시간을 측정하여 성능 차이를 확인합니다.

```python
# 다중 조건 필터링 예시: 리스트 컴프리헨션 vs. filter() 함수

# 1. 리스트 컴프리헨션 (다중 조건)
# 조건 1: 짝수 (x % 2 == 0)
# 조건 2: 3의 배수 (x % 3 == 0)
data = range(1000000)
# 다중 조건을 if and if 형태로 나열하는 것은 가독성을 저해할 수 있음
result_comp = [x for x in data if x % 2 == 0 if x % 3 == 0]

# 2. filter() 함수 (다중 조건)
# lambda 함수를 사용하여 가독성을 높일 수 있음
result_filter = list(filter(lambda x: x % 2 == 0 and x % 3 == 0, data))

# 3. 성능 비교 (timeit 모듈 사용)
import timeit

setup_code = """
data = range(1000000)
"""

# List comprehension: and 연산자로 조건을 통합하는 것이 일반적이며 더 효율적임
comprehension_code = "[x for x in data if x % 2 == 0 and x % 3 == 0]"

# filter() with lambda function
filter_code = "list(filter(lambda x: x % 2 == 0 and x % 3 == 0, data))"

# 벤치마크 실행 (number=100)
time_comp = timeit.timeit(comprehension_code, setup_code, number=100)
time_filter = timeit.timeit(filter_code, setup_code, number=100)

# print(f"List Comprehension Time: {time_comp:.6f} seconds")
# print(f"Filter Time: {time_filter:.6f} seconds")
```

### Insight

*   **성능:** 대규모 데이터 처리 시, 리스트 컴프리헨션이 `filter()` 함수보다 눈에 띄게 빠릅니다. 이는 `filter()` 함수가 각 요소에 대해 함수 호출 오버헤드(Function Call Overhead)를 발생시키는 반면, 리스트 컴프리헨션은 CPython 인터프리터 수준에서 최적화되어 구현되기 때문입니다.
*   **가독성:** `[x for x in data if a and b]` 형태로 조건을 묶으면 가독성과 성능을 동시에 확보할 수 있습니다. `filter()` 함수는 비개발자나 코드 입문자가 직관적으로 이해하기 쉬운 함수형 프로그래밍 스타일을 제공한다는 장점이 있습니다.
*   **결론:** 가독성이 최우선인 비즈니스 로직에서는 `filter()`를, 성능 최적화가 필요한 대규모 데이터 처리에서는 `List Comprehension`을 사용하는 것이 효과적입니다.
