### Context

파이썬에서 리스트 데이터를 처리할 때, 특정 조건을 만족하는 요소만 추출하는 필터링 작업은 매우 흔하다. 특히 여러 개의 조건을 적용해야 할 때, 가독성과 성능 사이에서 최적의 구현 방법을 선택하는 것이 중요하다.

`[x for x in list if cond1 if cond2]`와 같이 리스트 컴프리헨션(List Comprehension) 내부에 여러 개의 `if` 절을 사용하는 방식은 코드가 길어지거나 복잡해질 때 가독성이 떨어질 수 있다. 이 때문에 `filter()` 함수와 람다(lambda)식을 사용하여 `filter(lambda x: cond1 and cond2, list)` 형태로 코드를 작성하는 방법을 고려하게 된다.

### Core

파이썬에서 리스트 컴프리헨션과 `filter()` 함수는 기능적으로 동일한 결과를 얻을 수 있지만, 성능 측면에서 차이가 있다. 일반적으로 리스트 컴프리헨션이 `filter()` 함수를 사용하는 것보다 빠르다.

**1. 리스트 컴프리헨션 (List Comprehension)**

```python
# 여러 조건을 나열하는 방식 (가독성 저하 가능성)
data = list(range(100000))
filtered_data = [x for x in data if x > 1000 and x < 2000]
```

**2. filter() 함수와 람다식**

```python
# 람다식으로 조건을 결합하여 가독성 향상
filtered_data = list(filter(lambda x: x > 1000 and x < 2000, data))
```

**성능 비교**

리스트 컴프리헨션은 파이썬 인터프리터 레벨에서 C로 구현되어 있어 오버헤드가 적다. 반면, `filter()` 함수는 내부적으로 함수 호출(콜백 함수)을 필요로 하며, 람다 함수를 사용할 경우 매번 함수를 호출하는 데 따른 추가적인 오버헤드가 발생한다.

대규모 데이터셋에서 간단한 필터링 조건을 적용할 경우, 리스트 컴프리헨션이 `filter()` 함수보다 약 10%~20%가량 더 빠른 결과를 보여준다.

*   **성능 우위**: 리스트 컴프리헨션 (List Comprehension)
*   **성능 열위**: `filter()` 함수 (function call overhead)

### Insight

기술적 완성도를 위해 파이썬 코드를 작성할 때는 일반적으로 성능이 더 우수한 리스트 컴프리헨션을 우선적으로 고려하는 것이 좋다. 특히 대규모 데이터를 처리할 때는 성능 차이가 유의미하게 발생하므로, 컴프리헨션이 기본 선택지가 된다.

다만, 람다식 내에 복잡한 조건이나 다수의 논리 연산이 포함되어 가독성 확보가 중요할 때는 `filter()` 함수를 선택할 수 있다. 예를 들어, `lambda x: (cond1 and cond2) or cond3`와 같이 조건의 조합이 복잡해질 때 `filter()` 방식이 더 직관적일 수 있다.

**결론:** 파이썬의 핵심 철학 중 하나인 '가독성이 뛰어난 코드가 좋은 코드'라는 점을 염두에 두되, 성능이 중요한 경우 (예: 대용량 데이터 처리, 봇 자동화 작업) 리스트 컴프리헨션을 사용하고, 비전공자나 초보자가 읽기 쉬운 코드가 우선이라면 `filter()` 함수를 활용하는 것이 현명하다.
